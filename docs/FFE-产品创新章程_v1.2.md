## 一、项目背景 (Background)

### 1.1 商业机会

| 维度 | 事实 |
|------|------|
| **市场规模** | 中国 K12 英语教育市场超 ¥2,000 亿，游戏化学习赛道增速 >25%/年 |
| **政策窗口** | "双减"后线下培训收缩，线上轻量学习工具需求爆发 |
| **技术窗口** | Web 技术成熟（SPA、PWA、Service Worker），零分发成本 |
| **竞争空白** | 开源、轻量、可定制的教育工具市场尚未形成寡头 |

### 1.2 痛点洞察

**学生端**:
- 传统背单词/做练习枯燥，缺乏持续动力
- App 下载占用空间、家长管控严格、付费门槛高

**教师端**:
- 缺乏低成本、可定制的课堂互动工具
- 现有工具（如 Kahoot）需账号、需网络稳定、学习成本高

**家长端**:
- 无法判断孩子学习效果，监督成本高
- 市面产品质量参差不齐，缺少透明可审计的工具

### 1.3 为什么要现在做？

- 开源教育工具正处于早期爆发窗口期
- Web 游戏化学习尚无标杆产品
- 教师群体对数字化工具接受度已达拐点
- 技术栈（SPA + PWA + Cloudflare 边缘计算）已足够支撑零成本运营

## 二、概念原型 (Concept Prototype)

### 2.1 产品定义

**玛蒂尔达** 是一款基于 Electron 的桌面端英语学习游戏：
- **主推形态 — 桌面客户端**: Electron 打包，支持 macOS / Windows / Linux（v1.0 起）
- **辅助形态 — Web 版**: SPA 单页应用，部署于 GitHub Pages（体验入口）
- **核心玩法**: 闯关答题 — 随机抽题 → 即时反馈 → 积分连胜 → 下一关
- **技术架构**: Electron + Vue 3 + TypeScript + Vite，本地执行，支持离线使用
- **入口**: 桌面客户端本地安装（主入口）；Web 版浏览器直接访问（体验入口）

### 2.1.1 为什么选择 Electron 桌面客户端？

| 考量维度 | Web 版 | Electron 桌面版（主推） |
|----------|--------|------------------------|
| 软件分发 | ❌ 纯前端，无客户端感知 | ✅ 可执行文件，正规软件体验 |
| 离线可用 | ⚠️ 需 PWA 预缓存，体验受限 | ✅ 完全本地执行，零网络依赖 |
| 文件系统访问 | ❌ 浏览器沙箱受限 | ✅ 完整本地文件读写权限 |
| 本地 AI（Ollama） | ❌ CORS/跨域受限 | ✅ 直接调用本地 API，无网络延迟 |
| 数据持久化 | ❌ localStorage 容量受限 | ✅ 本地 SQLite/JSON 存储，容量无上限 |
| 自动更新 | ⚠️ 需刷新页面 | ✅ electron-updater 静默更新 |
| 分发渠道 | GitHub Pages | GitHub Releases + 应用商店 + 官网下载 |
| 家长/学校认可 | "网页游戏" | ✅ "正规学习软件"，可安装到系统 |
| 知识产权保护 | ❌ 源码完全暴露 | ✅ 代码打包，核心逻辑保护 |

**策略**：Electron 桌面版作为主推形态，Web 版仅作为体验入口。两者共用同一套前端代码库（Vite 构建），通过 Electron 的 `BrowserWindow` 加载本地打包后的 SPA，改造成本低，用户体验一致。

## 三、价值主张 (Value Proposition)

### 3.1 对学习者

> **"让英语学习像游戏一样有趣"**

| 价值维度 | 具体表现 | 竞品对比 |
|----------|----------|----------|
| 零门槛 | 打开浏览器即玩，无需注册/下载 | 多邻国需下载 App+注册 |
| 即时反馈 | 每题答完立刻知道对错 | 纸质练习无即时反馈 |
| 可重复 | 随机题库，每次体验不同 | 百词斩题库固定 |
| 成就感 | 关卡、积分、连胜、进度可视化 | 传统工具缺乏游戏化 |
| 隐私安全 | 匿名使用，不收集个人信息 | 多数 App 强制收集数据 |

### 3.2 对教师

> **"5 分钟创建课堂互动游戏"**

| 价值维度 | 具体表现 | 竞品对比 |
|----------|----------|----------|
| 可定制 | CSV 格式导入，Excel 即可编辑 | Kahoot 编辑需在线操作 |
| 零成本 | 免费开源，无订阅费用 | Kahoot 免费版功能受限 |
| 易部署 | GitHub Pages 托管，无需维护服务器 | 自建系统需运维 |
| 可共享 | 题库文件可复制分享给同事 | 多数工具不支持 |

### 3.3 对运营方

> **"零门槛 × 游戏化 × 开源社区 = 可持续增长飞轮"**

| 价值维度 | 具体表现 |
|----------|----------|
| 低成本获客 | 开源 + 内容营销 CAC < ¥3 |
| 低运营成本 | GitHub Pages 免费，Cloudflare 免费额度充足 |
| 社区共建 | 教师贡献题库，开发者贡献代码 |
| 数据驱动 | 匿名统计系统持续优化内容 |

## 四、目标与成功标准 (Objectives & Success Criteria)

### 4.1 北极星指标

**每周活跃学习者数 (WAL)** — 一周内完成 ≥ 1 关游戏的独立用户数

### 4.2 阶段目标

| 阶段 | 时间 | WAL | 题库 | 收入 | 关键里程碑 |
|------|------|-----|------|------|------------|
| 冷启动 | M1 | 100 | 500 题 | ¥0 | 推广化 README、首批内容、种子社群 |
| PMF 验证 | M3 | 500 | 500 题 | ¥0-500 | 1,000 UV、用户反馈报告、v1.1 发布 |
| 增长验证 | M6 | 2,000 | 1,000 题 | ¥2,000/月 | 5,000 UV、增值服务上线、首笔 B 端收入 |
| 规模期 | M12 | 8,000 | 3,000 题 | ¥10,000/月 | 20,000 UV、B 端客户、盈亏平衡 |
| 成熟期 | M18 | 15,000 | 5,000 题 | ¥30,000+/月 | 多收入来源稳定、社区自治 |

### 4.3 Go/No-Go 决策门

**M3 决策门**（满足 ≥ 3/4）:
- ☐ WAL ≥ 300
- ☐ 首关完成率 ≥ 50%
- ☐ 7 日留存 ≥ 10%
- ☐ 题库 ≥ 300 题

→ 通过：进入增长期，追加投入  
→ 未通过：2 周 Sprint 修复后再评估

**M6 决策门**（满足 ≥ 4/5）:
- ☐ WAL ≥ 1,000
- ☐ 7 日留存 ≥ 15%
- ☐ 月收入 ≥ ¥1,000
- ☐ B 端客户 ≥ 1
- ☐ GitHub Stars ≥ 150

→ 通过：启动 A 轮准备，规模化  
→ 未通过：重新评估商业模式

### 4.4 止损线

| 信号 | 阈值 | 触发时间 | 应对 |
|------|------|----------|------|
| 首关转化率过低 | < 15% | M3 | 重新审视 Landing 体验，A/B 测试 |
| 7 日留存过低 | < 5% | M3 | 深度用户访谈，定位留存障碍 |
| 收入为零 | M6 仍 ¥0 | M6 | 调整商业模式，探索新收入流 |
| 零外部贡献 | M6 仍 0 | M6 | 简化贡献流程，完善文档 |

### 10.3 产品定位确认（2026-06-12）

**市场部确认意见**（2026-06-12 10:21）：

基于 PRD v2.0、SRS v2.1 和产品验收报告，市场部对以下事项确认：

| 议题 | 确认内容 |
|------|----------|
| **产品定位** | 确认为「叙事驱动 + 英语学习」融合定位。以剧情冒险为外壳、英语学习为内核，让孩子觉得在玩、让家长看到在学 |
| **目标人群** | 收窄至 **10-13 岁**（小升初阶段，PET 水准），原 FFE 范围过宽（K12 教师+家长+自学者） |
| **核心玩法优先级** | 剧情系统 > 核心玩法小游戏（速读/拼写/记忆测试）> 关系系统（Miss Honey 信任度等）> 超能力系统（L6+ 意念控制） |
| **日常任务切入点** | Miss Honey 信任系统可扩展为 daily engagement 机制，支撑 FFE 要求的 7 日留存指标 |
| **意念控制机制** | PRD L6 设计为拖拽解谜小游戏：用"意念"（鼠标/触屏拖拽）移动物体到目标位置，越大越难控制，有能量条限制。适合 Web 实现 |

**建议**：产品规划部基于以上确认，更新 PRD v2.0 → v2.1，明确融合定位和优先级排布。
