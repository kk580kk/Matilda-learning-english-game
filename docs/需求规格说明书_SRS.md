#### 3.1.1 魔法书系统 (F-001)

**功能描述**：游戏首页展示魔法书动画，引导用户进入游戏。

**界面元素**：
| 元素 | 类型 | 说明 |
|------|------|------|
| 魔法书封面 | div | 书本封面样式，包含标题和装饰 |
| 打开按钮 | button | "✨ 打开魔法书" |
| 书页 | div | 左右两页，左页关卡，右页装备 |

**验收标准**：
- [ ] 魔法书封面显示正常
- [ ] 点击按钮播放打开动画
- [ ] 动画流畅，无卡顿
- [ ] 适配不同屏幕尺寸

#### 3.1.2 关卡系统 (F-002)

**功能描述**：关卡选择界面，展示关卡列表和进度状态。

**数据结构**：
```javascript
const levels = [
  {
    id: 1,
    name: "小试牛刀",
    difficulty: "简单",
    status: "unlocked", // locked | unlocked | completed
    song: "歌曲片段",
    questions: [...],
    boss: { name: "小怪物", hp: 50, emoji: "👹" }
  },
  // ...
];
```

**输入**：
- 用户点击关卡卡片

**处理**：
- 验证关卡解锁状态
- 加载关卡数据
- 进入游戏战斗界面

**输出**：
- 关卡详情展示
- 进入游戏界面

**界面元素**：
| 元素 | 类型 | 说明 |
|------|------|------|
| 关卡卡片 | div | 显示关卡编号、名称、难度 |
| 状态标识 | span | 锁定 🔒 / 解锁 ✅ / 已完成 🏆 |
| 关卡列表 | div | 网格布局，4 个关卡 |

**业务规则**：
1. 第一关默认解锁
2. 完成前一关解锁下一关
3. 已完成的关卡可以重复挑战
4. Boss 关卡需要完成所有前置关卡

**验收标准**：
- [ ] 关卡列表正确显示
- [ ] 锁定关卡无法进入
- [ ] 完成关卡后解锁下一关
- [ ] 关卡数据正确加载

#### 3.1.4 战斗系统 (F-004)

**功能描述**：答题与战斗的关联逻辑，攻击、受击、胜负判定。

**数据结构**：
```javascript
const battleState = {
  player: { hp: 100, maxHp: 100, attack: 10 },
  enemy: { name: "小怪物", hp: 50, maxHp: 50, attack: 15, emoji: "👹" },
  currentQuestion: 0,
  totalQuestions: 5,
  isPlayerTurn: true
};
```

**战斗流程**：
```
开始战斗 → 显示题目 → 玩家答题 → 判定结果 → 播放动画 → 检查胜负 → 下一题/结束
```

**伤害计算**：
| 答题结果 | 伤害值 | 动画效果 |
|----------|--------|----------|
| 答对 | 攻击力（基础 10） | 玩家攻击怪物 |
| 答错 | 怪物攻击力（15-25） | 怪物攻击玩家 |
| 完美跟读 | 攻击力 × 1.5 | 暴击特效 |

**胜负判定**：
- 胜利条件：怪物 HP ≤ 0
- 失败条件：玩家 HP ≤ 0
- 完成条件：答对所有题目

**界面元素**：
| 元素 | 类型 | 说明 |
|------|------|------|
| 玩家角色 | div | 玛蒂尔达形象 🧙‍♀️ |
| 怪物角色 | div | 怪物形象 |
| 玩家血条 | div | 显示当前 HP |
| 怪物血条 | div | 显示当前 HP |
| 伤害数字 | div | 飘字动画 |

**验收标准**：
- [ ] 血条显示正确
- [ ] 攻击动画流畅
- [ ] 伤害计算正确
- [ ] 胜负判定准确
- [ ] 战斗结束处理正确
